### 🎯 Next Proposal — #58: Autonomous Image Optimizer & WebP/AVIF Converter

**Problem:** Images often remain uncompressed or in inefficient formats (PNG/JPEG), bloating page weight and slowing LCP. No automated conversion to next-gen formats (WebP/AVIF) exists, missing free performance gains.

**Solution:** Weekly image optimizer:
- Scans `public/` and content image references (in frontmatter/body)
- Detects large/uncompressed images (`.png`, `.jpg` > threshold)
- Converts to WebP (lossless) and AVIF (lossy optimal) using `imagemin` or `sharp`
- Keeps originals as fallback for compatibility
- Generates `<picture>` elements with `srcset` for responsive loading
- Updates content frontmatter to point to optimized images (opt-in)
- Weekly report: bytes saved, count optimized, compression ratio

**Components:**
- `automation/image-optimizer.cjs` — image detection; conversion; HTML/md update (careful frontmatter handling)
- `.github/workflows/image-optimizer.yml` — Weekly Thursday 12:00 UTC + manual
- `docs/IMAGE-OPTIMIZER.md`
- State: `.hermes/memory/image-optimizer/` (inventory.json, optimized.json, stats.json)

**Config (env):**
```bash
IMAGE_OPTIMIZER_INPUT_DIRS=public,app/blog/images,app/services/images
IMAGE_OPTIMIZER_MAX_SIZE_BYTES=500000  # optimize if >500KB
IMAGE_OPTIMIZER_FORMATS=webp,avif
IMAGE_OPTIMIZER_LOSSLESS=true          # WebP lossless
IMAGE_OPTIMIZER_AUTO_UPDATE=false      # safe default; manual approval for content changes
```

**Safety:** Read-only scan; writes only to `public/optimized/` or alongside originals with backup; content file updates only if AUTO_UPDATE=true; deterministic compression; preserves originals.

**Standing:** Ready for autonomous implementation.

### 🎯 Next Proposal — #60: Autonomous Lazy-Load Priority Scroller

**Problem:** Performance gains from lazy-loading are maximized when above-the-fold images load eagerly. Currently, all images may be lazy-loaded equally, delaying critical content.

**Solution:** Weekly analyzer that:
- Parses each page route; determines first contentful paint (FCP) candidate images (hero, logo, above-fold)
- Adds `fetchpriority="high"` and `loading="eager"` to those images
- Ensures `<link rel="preload" as="image">` for top hero image
- Verifies below-fold images have `loading="lazy"`
- Opens GitHub PR with automatic annotations to optimize LCP

**Components:**
- `automation/lazyload-priority-scroller.cjs` — HTML/TSX parser; identifies critical images; edits files safely (preserves formatting)
- `.github/workflows/lazyload-priority-scroller.yml` — Weekly Friday 04:00 UTC; PR creation
- `docs/LAZYLOAD-PRIORITY-SCROLLER.md`
